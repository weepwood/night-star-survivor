# Night Star Survivor

一款以《吸血鬼幸存者》式自动战斗为基础、采用暗夜像素幻想视觉风格的纯前端网页游戏。

## 当前内容

- 三名属性与初始能力不同的守夜者
- 七种武器：星矢阵列、月轮刃、寒星领域、星雷链、彗星权杖、棱镜星轮、虚空星雷
- 多类遗物与强化，包括星云透镜、时隙碎片、微型黑洞、凤凰余烬和星迹披风
- 星陨天灾、无尽棱镜、黑洞雷场等新武器进化
- 六种战场道具：星露药剂、新星电池、引力棱镜、日蚀炸弹、星幕护盾、星蚀宝箱
- 普通、精英、远程和首领敌人
- 阶段切换、首领警报和地面灾害
- 冲刺、星爆、夜潮超载主动技能
- 像素粒子、彗星预警、爆炸残痕、进化震波、精英标记和动态星尘环境
- 基于 Web Audio 的轻量合成音效

## Nightfall V3 战斗驾驶舱

- 原本分散在顶部和左右两侧的信息被重组为战斗驾驶舱，不再让多块常驻面板同时遮挡战场
- 顶部战术条只保留阶段、时间、当前目标、敌群压力、首领生命和首领破韧状态
- 底部技能坞集中显示冲刺、夜潮超载和星爆状态，并支持鼠标点击或键盘操作
- 左侧采用“构筑 / 星契”标签切换，右侧采用“输出 / 道具 / 图鉴”标签切换
- 小窗口会继续压缩信息，只保留角色生存、首领状态和主动技能
- 开局界面改为守夜核心选择，展示角色定位、火力、机动和生存倾向
- 胜利或失败后显示夜幕评级、处决积分、超载次数、首领破韧、连锁击杀和星蚀遗产数量

## 夜潮超载与连锁处决

- 造成实际伤害、击杀敌人、击败精英和击破首领韧性都会积累夜潮
- 夜潮达到 100% 后可按 `E` 进入至少 8 秒的夜潮超载
- 超载期间造成伤害提高 55%，受到伤害降低 22%，冲刺恢复加快
- 超载启动时会清除附近敌方弹幕并提供短暂无敌，形成主动反攻窗口
- 超载期间击杀敌人会触发范围裂变爆炸，爆炸最多影响 8 个目标
- 连锁爆炸递归深度限制为两层，保留大规模清场爽感，同时避免无限递归和性能失控
- 星爆在超载期间释放会延长 0.8 秒超载时间，形成主动技能之间的连携
- 长时间脱离战斗后，未充满的夜潮会缓慢衰减，鼓励持续进攻而不是绕场拖延

## 首领破韧与星蚀遗产

- 对首领持续造成实际伤害会积累破韧值
- 破韧充满后首领停止移动与攻击 3 秒，并清除场上敌方弹幕
- 破韧期间对首领造成的伤害额外提高 45%，形成明确的爆发时机
- 破韧结束后首领会恢复原始移动速度与攻击节奏
- 击败前两名首领后，会通过实时非暂停升级终端提供一项星蚀遗产选择
- 当前包含裂变引擎、奇点心脏、猩红反应堆、水银星跃、临界质量和残光协议
- 星蚀遗产可以强化超载持续时间、连锁爆炸、暴击夜潮、冲刺夜潮、低生命反击或击杀恢复
- 最终首领死亡会优先进入胜利结算，不会再弹出遗产选择或与失败结算竞争

## 夜幕任务与胜利条件

- 每局目标是击败三名星蚀首领
- 血月监视者、星蚀巨像和无光祭司分别在约 40、80、120 秒降临
- 顶部战术条持续显示下一名首领倒计时、首领生命和阶段目标
- 击败第三名首领后立即进入独立胜利结算
- 胜利结算显示用时、等级、击杀数、本局星契和 Nightfall V3 战斗评级
- 角色死亡仍进入失败结算

## 随机星契组合

- 每次开始游戏会从星契池随机抽取三条本局组合路线
- 星契不会直接生效，需要玩家获得对应武器、遗物或强化后才能激活
- 当前包含星雷穿刺、冻月回响、引力天灾、棱镜风暴、凤凰黎明、时隙猎杀、猩红月环和星云聚焦
- 激活后的星契会实际影响伤害、承伤、击退、击杀回血、星爆获取、冲刺恢复或连击保持时间
- 左侧星契标签页显示组合条件、奖励和当前 `LOCKED / ONLINE` 状态

## 星象契约与星印共鸣

- 25、65、105 秒会出现一次不暂停战斗的星象契约
- 每次从随机任务池提供三项风险任务，使用 `4`、`5`、`6` 快速选择
- 契约包含限时猎杀、输出挑战、无伤航路、星尘采集、精英悬赏、冲刺训练和移动距离等目标
- 风险会临时提高新生成敌人的生命、速度或玩家承伤，失败后星潮威胁提高 10 秒
- 完成契约会获得永久星印和对应奖励，例如伤害、暴击、护甲、生命、磁吸、恢复、移动速度或冲刺冷却
- 两枚同类型星印会触发猎杀、守望、采集、跃迁或爆发共鸣，并获得额外永久加成
- 连续击杀会形成星链倍率：10、25、50 连锁分别提高本局实时伤害倍率
- 升级构筑出现时契约选择倒计时会让位，避免两个选择面板同时抢占操作

## 音效与打击反馈

- Web Audio 增加动态压缩与总线控制，降低多音效同时播放时的刺耳和爆音
- 射击、普通命中、暴击、击杀、精英击杀、拾取、升级、冲刺、星爆、首领、目标完成、胜利和失败使用不同音层
- 高频射击与命中音效增加节流，避免敌人密集时形成持续噪声
- 普通命中使用局部十字像素冲击，不进行全屏闪白
- 暴击使用更强局部冲击和独立音效
- 夜潮就绪、超载启动、首领破韧和星蚀遗产接入拥有独立战斗播报与像素冲击
- 普通敌人受到命中时会产生轻微击退，首领不会被普通击退
- 角色周围显示分段生命、护甲、冲刺就绪和星爆就绪像素

## 实时非暂停构筑

- 升级选择期间游戏不会暂停，而是把敌人、弹幕、生成和危险区降至 35% 世界流速
- 玩家移动、冲刺和星爆仍可正常操作，武器计时保持约 60% 速度
- 升级终端固定在战场侧边，不再使用全屏遮罩
- 每次选择提供 8 秒倒计时，超时自动选择系统推荐项
- 支持数字键 `1`、`2`、`3` 快速选择三张强化卡
- 升级首次弹出时击退附近敌人，并提供 1.2 秒、70% 减伤保护
- 连续升级使用队列处理，不会丢失选择，也不会反复叠加击退和减伤
- 推荐逻辑优先完成武器进化，其次强化已有构筑，并根据当前生命值调整生存选项权重
- 手动暂停仍会冻结战斗、升级倒计时和保护计时
- 游戏结束或重新开始时会清空未处理的升级队列

## 自适应详情与信息层级

- 战斗驾驶舱默认只展示当前需要作出决策的信息
- 武器说明、星契条件、道具用途、升级描述和契约风险通过鼠标悬停或键盘聚焦查看
- 提示框会根据目标位置自动选择左右方向，并限制在浏览器可视区域内
- 动态生成的道具、升级卡、契约卡、星印和首领遗产会自动接入详情系统
- 左右标签页保证每侧同时只展示一类详细信息，减少战场遮挡

## 像素战斗统计与稳定显示

- 输出标签页展示最近 5 秒 DPS、总伤害、峰值 DPS 和有效命中次数
- 本局道具标签页记录获得的武器、遗物、强化、战场掉落和武器进化
- 左下重复的最近道具条已关闭，本局道具只在右侧标签页展示
- HUD 更新频率受到限制，减少 DOM 重排和文字跳动
- 普通命中不再触发明显全屏抖动，强反馈主要使用局部像素震波和短促声音层
- 限制全屏闪白透明度，并自动遵循系统的“减少动态效果”设置

## 单屏 Pixel-perfect 渲染

- 战场逻辑分辨率：`640 × 360`
- 中央画面就是全部可移动地图，不存在镜头外隐藏区域
- 摄像机固定在 `(0, 0)`，玩家、敌人、首领、危险区与掉落均限制在竞技场边界内
- 所有主要角色、敌人、弹道和粒子均按整数像素绘制
- Canvas 禁用图像平滑，显示层使用最近邻缩放
- 页面根据窗口宽度、高度和 `devicePixelRatio` 优先使用整数物理像素倍率

## 操作

- `WASD` / 方向键：移动
- `Space`：冲刺
- `E`：夜潮充满后启动超载
- `Q`：星爆
- `1` / `2` / `3`：升级时快速选择
- `4` / `5` / `6`：星象契约快速选择
- `P` / `Esc`：暂停

## 本地运行

这是一个纯静态项目，可直接打开 `index.html`，也可以使用任意静态服务器：

```bash
python -m http.server 8080
```

随后访问 `http://localhost:8080`。

## 部署

仓库包含 GitHub Pages 工作流。修改合并到 `main` 后，会自动发布静态站点。
