# Night Star Survivor

一款以《吸血鬼幸存者》式自动战斗为基础、采用暗夜像素幻想视觉风格的纯前端网页游戏。

## 当前内容

- 三名属性与初始能力不同的守夜者
- 七种武器：星矢阵列、月轮刃、寒星领域、星雷链、彗星权杖、棱镜星轮、虚空星雷
- 多类遗物与强化，包括星云透镜、时隙碎片、微型黑洞、凤凰余烬和星迹披风
- 星陨天灾、无尽棱镜、黑洞雷场等新武器进化
- 六种战场道具：星露药剂、新星电池、引力棱镜、日蚀炸弹、星幕护盾、星蚀宝箱
- 普通、精英、远程和首领敌人
- 阶段切换、首领警报和地面灾害
- 冲刺、星爆与夜潮超载主动技能
- 像素粒子、彗星预警、爆炸残痕、进化震波、精英标记和动态星尘环境
- 基于 Web Audio 的轻量合成音效

## Nightfall V3 战斗驾驶舱与夜潮超载

- 顶部战术条集中显示阶段、时间、目标、敌群压力、首领生命与破韧进度
- 底部技能坞集中显示冲刺、夜潮超载和星爆状态
- 左侧通过“构筑 / 星契”标签切换，右侧通过“输出 / 道具 / 图鉴”标签切换
- 实际伤害、击杀、精英、首领和破韧会积累夜潮
- 夜潮达到 100% 后按 `E` 进入至少 8 秒超载，获得伤害提高、承伤降低和冲刺恢复加速
- 超载期间击杀敌人会触发最多两层、每层最多影响 8 个目标的裂变爆炸
- 对首领持续输出会积累破韧，破韧后获得 3 秒、额外 45% 伤害的处决窗口
- 击败前两名首领后可选择改变后半局玩法的星蚀遗产
- 胜负结算显示夜幕评级、处决积分、超载次数、破韧次数和连锁击杀

## 夜幕任务与胜利条件

- 每局目标是击败三名星蚀首领
- 血月监视者、星蚀巨像和无光祭司分别在约 40、80、120 秒降临
- 目标面板持续显示下一名首领倒计时、首领生命和阶段完成状态
- 击败第三名首领后立即进入独立胜利结算
- 胜利结算显示用时、等级、击杀数和本局激活的星契组合
- 角色死亡仍进入原有失败结算

## 随机星契组合

- 每次开始游戏会从星契池随机抽取三条本局组合路线
- 星契不会直接生效，需要玩家获得对应武器、遗物或强化后才能激活
- 当前包含星雷穿刺、冻月回响、引力天灾、棱镜风暴、凤凰黎明、时隙猎杀、猩红月环和星云聚焦
- 激活后的星契会实际影响伤害、承伤、击退、击杀回血、星爆获取、冲刺恢复或连击保持时间
- 左侧星契终端会显示组合条件、奖励和当前 `LOCKED / ONLINE` 状态

## 星象契约与星印共鸣

- 25、65、105 秒会出现一次不暂停战斗的星象契约
- 每次从随机任务池提供三项风险任务，使用 `4`、`5`、`6` 快速选择
- 契约包含限时猎杀、输出挑战、无伤航路、星尘采集、精英悬赏、冲刺训练和移动距离等目标
- 风险会临时提高新生成敌人的生命、速度或玩家承伤，失败后星潮威胁提高 10 秒
- 完成契约会获得永久星印和对应奖励，例如伤害、暴击、护甲、生命、磁吸、恢复、移动速度或冲刺冷却
- 两枚同类型星印会触发猎杀、守望、采集、跃迁或爆发共鸣，并获得额外永久加成
- 连续击杀会形成星链倍率：10、25、50 连锁分别提高本局实时伤害倍率
- 升级构筑出现时契约选择倒计时会让位，避免两个选择面板同时抢占操作
- 星尘强化终端出现时，星象契约自动折叠到左上角；极小窗口会暂时隐藏契约终端

## 长局性能稳定

- 敌人数量使用随阶段、等级和难度变化的动态预算，普通敌人达到预算后暂停生成
- 首领始终允许生成，紧急情况下优先清理普通敌人，不会误删首领和精英
- 投射物碰撞由“每颗弹丸遍历全部敌人”改为 56 像素空间网格候选查询
- 玩家弹丸、敌方弹幕、粒子、伤害数字、危险区和光束均设置明确实体预算
- 过量星尘会合并到保留的星尘中，不会因为性能保护丢失经验值
- 性能守卫位于所有战斗包装层最外侧，同时覆盖普通战斗、实时升级、契约和 Nightfall V3
- 自动检查包含 180 秒无击杀压力模拟，确保实体数量不会越过动态预算和硬上限

## 音效与打击反馈

- Web Audio 增加动态压缩与总线控制，降低多音效同时播放时的刺耳和爆音
- 射击、普通命中、暴击、击杀、精英击杀、拾取、升级、冲刺、星爆、首领、目标完成、胜利和失败使用不同音层
- 高频射击与命中音效增加节流，避免敌人密集时形成持续噪声
- 普通命中使用局部十字像素冲击，不进行全屏闪白
- 暴击使用更强局部冲击和独立音效
- 普通敌人受到命中时会产生轻微击退，首领不会被击退
- 角色周围显示分段生命、护甲、冲刺就绪和星爆就绪像素
- 左侧角色状态显示护甲、冲刺、星爆和连击信息

## 实时非暂停构筑

- 升级选择期间游戏不会暂停，而是把敌人、弹幕、生成和危险区降至 35% 世界流速
- 玩家移动、冲刺和星爆仍可正常操作，武器计时保持约 60% 速度
- 升级终端固定在战场侧边，不再使用全屏遮罩
- 每次选择提供 8 秒倒计时，超时自动选择系统推荐项
- 支持数字键 `1`、`2`、`3` 快速选择三张强化卡
- 升级首次弹出时击退附近敌人，并提供 1.2 秒、70% 减伤保护
- 连续升级使用队列处理，不会丢失选择，也不会反复叠加击退和减伤
- 推荐逻辑优先完成武器进化，其次强化已有构筑，并根据当前生命值调整生存选项权重
- 手动暂停仍会冻结战斗、升级倒计时和保护计时
- 游戏结束或重新开始时会清空未处理的升级队列

## 自适应精简 HUD

- 默认启用精简模式，左右 HUD 宽度、顶部指标、升级终端和星象契约终端都会缩小
- 生命、经验、星爆、任务、DPS 等战斗必要信息保持常驻
- 武器说明、星契条件、道具用途、升级描述和契约风险不再长期占用画面
- 鼠标悬停或键盘聚焦对应项目时，会在独立像素提示框中显示完整详情
- 提示框会根据目标位置自动选择左右方向，并限制在浏览器可视区域内
- 动态生成的道具、升级卡、契约卡和星印会自动接入悬停详情
- 顶部提供 `HUD：精简 / HUD：展开` 按钮，可随时切换完整显示
- 小窗口继续隐藏次要面板，只保留必要战斗状态，避免 UI 覆盖竞技场

## 像素战斗统计与稳定显示

- 顶部持续显示最近 5 秒 DPS 和本局累计总伤害
- 输出终端展示当前 DPS、总伤害、峰值 DPS 和有效命中次数
- 本局道具终端记录获得的武器、遗物、强化、战场掉落和武器进化
- 左下重复的最近道具条已关闭，本局道具只在右侧终端展示
- 关键 HUD 节点直接写入 HTML，不再完全依赖 JavaScript 动态创建
- 资源 URL 使用版本参数，避免 GitHub Pages 或浏览器继续使用旧版 HUD 文件
- 低高度和窄窗口不会再隐藏全部输出信息，而是切换成紧凑四项统计终端
- HUD 更新频率限制为 10Hz，输出统计限制为 5Hz，减少 DOM 重排和文字跳动
- 普通命中不再触发明显全屏抖动，强反馈震动限制在 1～2 个逻辑像素
- 取消角色受伤时的整帧隐身闪烁和敌人的整身白闪，改用稳定的像素定位标记
- 限制全屏闪白透明度，并自动遵循系统的“减少动态效果”设置
- HUD、按钮、卡片、血条、道具与输出终端统一使用硬边像素风格

## 全窗口战场 HUD

- 网页窗口本身就是完整游戏界面，不再保留页面外层空白和独立状态区域
- Canvas 根据窗口宽高自动选择最大可容纳的完整显示尺寸
- HUD 使用 Canvas 实际边界计算安全区域，不会漂移到 16:9 战场之外
- 浏览器缩放、窗口变化和横竖屏切换时会实时重新计算 HUD 锚点
- 顶部显示时间、等级、击杀、阶段、威胁、夜幕目标和输出数据
- 左侧显示角色状态、生命、经验、星爆、任务进度、当前构筑和随机星契
- 右侧显示输出统计、本局道具和战场道具图鉴
- 小窗口会自动隐藏次要说明并压缩目标与星契信息

## 单屏 Pixel-perfect 渲染

- 战场逻辑分辨率：`640 × 360`
- 中央画面就是全部可移动地图，不存在镜头外隐藏区域
- 摄像机固定在 `(0, 0)`，玩家、敌人、首领、危险区与掉落均限制在竞技场边界内
- 所有主要角色、敌人、弹道和粒子均按整数像素绘制
- Canvas 禁用图像平滑，显示层使用最近邻缩放
- 页面根据窗口宽度、高度和 `devicePixelRatio` 优先使用整数物理像素倍率

## 操作

- `WASD` / 方向键：移动
- `Space`：冲刺
- `E`：夜潮超载
- `Q`：星爆
- `1` / `2` / `3`：升级时快速选择
- `4` / `5` / `6`：星象契约快速选择
- `P` / `Esc`：暂停

## 本地运行

这是一个纯静态项目，可直接打开 `index.html`，也可以使用任意静态服务器：

```bash
python -m http.server 8080
```

随后访问 `http://localhost:8080`。

## 部署

仓库包含 GitHub Pages 工作流。修改合并到 `main` 后，会自动发布静态站点。
